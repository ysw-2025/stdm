# STDM 门店经营触点诊断地图

> 顾客每走一步，门店在哪个触点漏掉流量、信任、转化、体验与复购。  
> 15 分钟，找到门店真正漏钱的地方。

**STDM（Store Touchpoint Diagnostic Map）** 是由「位道策划 · 杨首位」原创的餐饮门店第三方诊断工具，面向连锁品牌的 SI 空间标准化设计与门店经营触点诊断。

## 在线访问

- **主工具**：https://ysw-2025.github.io/stdm/  
  7 站位 × 32 触点 × 三态清单，完成诊断后导出小红书图文数据。
- **小红书图文预览**：https://ysw-2025.github.io/stdm/xhs_preview.html  
  浏览器内生成 3:4 小红书卡片 PNG 与诊断报告 PDF。

## 当前版本

- 主工具：`v2.6.33`
- 小红书报告页：`v2.6.38`

## 核心功能

- 顾客旅程 6 段漏斗：看得见 → 找得着 → 等得住 → 进得来 → 吃得好 → 会再来
- 7 大经营系统：品牌 / 产品 / 空间 / 服务 / 运营 / 信任 / 关系
- 6 类损耗：流量 / 转化 / 体验 / 效率 / 信任 / 复购
- 三态评估：✓ 合格 / ⚠ 预警 / ✗ 漏钱
- 权重计分 + 食安红线熔断（健康分封顶 60）
- 双雷达对比：本店 vs 行业基准 80
- 门店实拍图上传、放大、删除
- JSON 诊断包导入 / 导出
- 小红书 3:4 卡片自动生成（PNG ZIP + PDF 诊断报告）
- 无图触点智能合并排版

## 仓库说明

本仓库是 **GitHub Pages 固定部署仓库**，用于托管线上访问版本。主要开发文件为本地单文件 HTML：

- `门店经营触点诊断地图_STDM_v2.html`（主工具）
- `xhs_preview.html`（小红书图文预览）

更新流程：修改本地主文件 → 复制到本仓库 → `md5` 校验一致 → `git push` → GitHub Pages 自动刷新（1–2 分钟）。

## 使用方式

1. 浏览器打开主工具链接，填入品牌、品类、诊断人信息。
2. 按 7 站位逐项检查，标记 ⚠ 预警 / ✗ 漏钱，上传门店照片。
3. 完成诊断后，点击「导出小红书图文」保存数据。
4. 打开小红书预览链接，生成并下载 PNG 卡片与 PDF 报告。

> **注意**：GitHub Pages 域名（`github.io`）在中国大陆部分地区可能无法直接访问。国内客户建议使用本地单文件分发或国内托管方案。
